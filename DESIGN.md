# DESIGN.md

## 项目定位

**哄哄模拟器** — 一款仿微信聊天界面的 AI 互动应用。用户选择"女朋友"或"男朋友"角色后，AI 扮演该角色因某个场景生气（怒气值 100），用户通过自由输入对话来安抚对方，将怒气值降到 0 即为胜利。男女通用。

## 产品首页风格 — Apple 设计语言

### 气质与意象
- **意象锚点**：深夜床头，手机屏幕柔光映在脸上，置顶聊天框微微亮着，指尖悬在键盘上斟酌措辞
- **核心情绪**：克制、温暖、高级——像 Apple 产品页那样干净利落地讲一个"哄人"的故事
- **差异化**：不做卡通可爱风、不做游戏化UI，追求「这看起来像是一个 Apple 原生应用」的错觉

### 字体排版
- **首页标题**：`-apple-system, BlinkMacSystemFont, "SF Pro Display", "Helvetica Neue", sans-serif`，Apple 系统字体栈
- **正文字体**：`-apple-system, BlinkMacSystemFont, "SF Pro Text", "Helvetica Neue", sans-serif`
- **字号层级**：
  - Hero 标题：`56px` / `3.5rem`（约 7 个字内换行）
  - 区块标题：`32px` / `2rem`
  - 卡片标题：`17px` / `1.0625rem`
  - 正文/描述：`14px` / `0.875rem`
  - 辅助文字：`12px` / `0.75rem`

### 布局结构
- **页面流**：Hero 全屏 → Features 三列 → How it Works 时间线 → Start 选择面板 → 极简 Footer
- **间距节奏**：section 间距 120px，内边距统一 24px
- **最大宽度**：内容区 `980px`（Apple 标准宽度），卡片区可放宽到 `1024px`
- **圆角**：大区块 `20px`，卡片 `16px`，按钮 `14px`，极小元素 `8px`

### 动效与交互
- **英雄区**：标题渐入 + 图片轻微浮起（hover 视差）
- **卡片**：hover 时微微上浮（`translateY(-2px)`）加阴影增强
- **选择组件**：点击/选中时 spring 弹性动画（`active:scale-[0.97]`）
- **滚动**：平滑滚动，section 进入时淡入上滑
- **全部动效**：`transition-all duration-300 ease-out`

### 设计禁忌
- ❌ 禁止大面积纯黑/纯白背景（用 `#f5f5f7` / `#1c1c1e` 替代纯白/纯黑）
- ❌ 禁止使用 emoji 作为功能性 icon（仅装饰性使用）
- ❌ 禁止圆角卡通风格、粗重阴影或过于花哨的渐变
- ❌ 禁止中文衬线字体
- ❌ 禁止多余花哨的装饰线——用留白代替分割线
